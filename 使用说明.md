# ⚡ 快速开始 - 电竞选手数据分析面板

## 🔍 重要提示

**直接打开 HTML 文件无法显示数据！** 这是因为浏览器的安全策略限制。

### ✅ 正确的打开方式（3 选 1）

#### 方法 1：使用 VS Code 的 Live Server 插件（最简单）
1. 安装 VS Code 的 "Live Server" 插件
2. 右键 `index.html` → "Open with Live Server"
3. 浏览器会自动打开页面

#### 方法 2：使用 Python 启动本地服务器
```bash
# 打开命令行，进入项目目录
python -m http.server 8080
```
然后访问：http://localhost:8080/index.html

#### 方法 3：使用 Node.js 的 http-server
```bash
# 安装 http-server
npm install -g http-server

# 进入项目目录，运行
http-server -p 8080
```
然后访问：http://localhost:8080/index.html

### 🧪 测试数据加载
双击打开 [`test-data.html`](file:///D:/Desktop/CivilWar/test-data.html) 可以检测 JSON 文件是否能正常加载。

## 🎯 功能说明

本系统会**自动加载所有 JSON 文件**并综合分析每名选手的多场比赛数据。

### 主要特点：
- ✅ 自动扫描并加载所有 JSON 比赛数据
- ✅ 综合统计每名选手的场均表现
- ✅ 追踪选手的比赛历程
- ✅ 多维度数据可视化分析

## 📁 如何添加新比赛数据

### 步骤 1：准备 JSON 文件
将你的比赛数据 JSON 文件（如 `0315_143022.json`）放入项目目录。

### 步骤 2：修改 app.js
打开 [`app.js`](file:///D:/Desktop/CivilWar/app.js) 文件，找到第 9-14 行：

```javascript
// 第 9-14 行
const DATA_FILES = [
    '0314.json',           // 👈 现有文件
    '0311_205642'     // 👈 现有文件
    // 👇 在这里添加新的 JSON 文件名
];
```

添加新文件的名称：

```javascript
const DATA_FILES = [
    '0314.json',
    '0311_205642',
    '0315_143022.json'  // 👈 新增这一行
];
```

### 步骤 3：刷新页面
保存文件并刷新浏览器，系统会自动综合分析所有数据。

## 📊 数据分析维度

### 个人数据统计
- **总比赛场次** - 参赛次数
- **平均评分** - 场均表现分
- **场均 KDA** - (击杀 + 助攻) / 死亡
- **胜率** - 获胜场次百分比
- **总击杀/助攻/伤害/经济**
- **最佳/最差比赛** - 单场表现极值
- **比赛历程** - 每场比赛的详细数据

### 可视化图表
1. **总伤害对比** - 柱状图
2. **总经济对比** - 柱状图  
3. **平均评分雷达图**
4. **场均 KDA 趋势图**

## 🔧 常见问题

### Q: 为什么页面只显示框架，没有数据？
**A:** 这是因为浏览器的安全策略（CORS）限制。直接打开 HTML 文件时，JavaScript 无法读取本地 JSON 文件。

**解决方案：**
1. ✅ 使用 VS Code 的 Live Server 插件（最简单）
2. ✅ 使用 Python 启动本地服务器：`python -m http.server 8080`
3. ✅ 使用 Node.js 的 http-server：`http-server -p 8080`
4. ✅ 部署到 GitHub Pages（上线后就没有这个问题了）

### Q: 如何检测数据是否能正常加载？
**A:** 双击打开 [`test-data.html`](file:///D:/Desktop/CivilWar/test-data.html) 文件，它会自动测试所有 JSON 文件并显示详细结果。

### Q: 数据加载失败怎么办？
**解决方法：**
1. 确保按照上述方法启动了本地服务器
2. 检查浏览器控制台错误信息（按 F12）
3. 确认 JSON 文件格式正确
4. 使用 test-data.html 诊断问题

### Q: 如何查看某名选手的详细数据？
A: 系统会自动合并同名选手的所有比赛数据，显示其综合统计和比赛历程。

### Q: 可以只分析特定位置的选手吗？
A: 使用页面上方的位置筛选器即可。

## 💡 提示

1. **JSON 文件命名**：建议使用 `日期_时间.json` 格式，便于管理
2. **数据量控制**：建议单个 JSON 文件不超过 5MB
3. **选手 ID**：系统使用游戏 ID（nickName）作为唯一标识，同名会自动合并

## 🚀 部署到 GitHub Pages

1. 将所有文件上传到 GitHub 仓库
2. 在 Settings → Pages 中启用 GitHub Pages
3. 访问生成的链接即可
